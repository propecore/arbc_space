Техническая документация (AI_Technology_165)

1.Назначение проекта

Проект arbc.space* — это лёгкий, бесплатный и полностью открытый веб-инструмент, созданный для быстрого ознакомления жителей городов и путешественников с оценкой трёх типов рисков, имеющих связь с космическими и климатическими процессами. Основные категории рисков:
· вероятность возникновения крупных пожаров (лесных, степных, городских) в ближайшие недели/месяцы,
· риск значительных наводнений и паводков,
· уровень опасности геомагнитных бурь (влияние на энергосистемы, спутники, радиосвязь, здоровье метеозависимых людей).
Проект решает задачу демократизации доступа к агрегированным научным данным NASA и NOAA, избавляя обычного пользователя от необходимости самостоятельно скачивать, обрабатывать и интерпретировать огромные датасеты (FIRMS, GPM/IMERG, SWPC Kp/Dst и др.). Целевая аудитория очень широкая:
жители крупных городов, интересующиеся экологической и космической безопасностью,
туристы, планирующие поездки в регионы с высоким риском стихийных бедствий,
школьники и студенты, изучающие географию, экологию, астрономию, космическую погоду,
популяризаторы науки и блогеры, которым нужен простой и красивый инструмент для демонстрации связи «космос → Земля».
*Проект не является официальной системой раннего предупреждения и не заменяет прогнозы МЧС, гидрометцентров или специализированных служб. Он носит исключительно образовательный и информационно-справочный характер.


2.Описание продукта

arbc.space* — классическое одностраничное веб-приложение (Single Page Application light), не использующее серверную часть для обработки пользовательских запросов в реальном времени. Визуальная структура страницы (сверху вниз):
1.Большой заголовок «Астрономические риски»
2.Краткое 1–2-строчное пояснение о происхождении данных и о том, что это не прогноз, а усреднённая оценка
3.Большой кликабельный список городов (на февраль 2026 — 15 городов)
4.Пустая область результата с текстом-заглушкой «Результат будет отображён здесь после выбора города.»
5)Футер с копирайтом, ссылкой на GitHub (если репозиторий публичный), датой последнего обновления данных
После выбора города область результата мгновенно заменяется шестью карточками / строками:
· Название города, текущая погода (в градусах Цельсия), фаза Луны
· Пожары — Низкий / Средний / Высокий риск + цветовая индикация
· Наводнения и Геомагнитные бури — аналогично
*Приложение не сохраняет состояние между перезагрузками страницы, не использует LocalStorage / Cookies, не отправляет никаких данных на сервер. Всё вычисление и отображение происходит исключительно в браузере пользователя после загрузки страницы.

3.Требования

Функциональные:
· Список городов должен содержать минимум 15–20 населённых пунктов с разных континентов
· Поддержка кириллицы и латиницы без искажений
· Мгновенная (≤ 400 мс) смена содержимого после клика
· Адаптивная вёрстка (мобильные 320–480 px, планшеты, десктоп)
· Возможность открыть сайт без интернета после первого посещения (если реализован Service Worker — опционально)
Нефункциональные:
· Время первой загрузки страницы ≤ 2.8–3.2 с при 4G-соединении
· Размер страницы (HTML + CSS + JS + данные) ≤ 280–340 КБ
· Поддерживаемые браузеры: Chrome 114+, Firefox 110+, Safari 16.4+, Edge 114+
· Uptime хостинга ≥ 99.6%
· Отсутствие внешних трекеров (Google Analytics, Meta Pixel и т.п.)
· Обязательные заголовки безопасности:
oContent-Security-Policy (minimal)
oX-Content-Type-Options: nosniff
oX-Frame-Options: DENY
oReferrer-Policy: strict-origin-when-cross-origin
Ограничения и нефункциональные допущения
· Данные обновляются вручную (или через GitHub Actions раз в 1–7 дней)
· Нет поддержки добавления пользователем своего города
· Нет авторизации, комментариев, рейтинга, обратной связи

4.Архитектура / схема работы

Браузер пользователя
        ↓
Статический хостинг (Vercel / Netlify / Cloudflare Pages)
        ↓
index.html  ←  style.css  ←  main.js (все данные внутри)
        ↓
Пользователь кликает по городу
        ↓
JavaScript функция → берёт данные из объекта → формирует HTML → вставляет в DOM
Варианты развития (возможные следующие шаги):
1.Данные остаются захардкоженными → самый быстрый и надёжный вариант
2.Данные лежат в отдельном JSON-файле на том же хостинге → fetch один раз
3.Данные обновляются автоматически раз в сутки через GitHub Actions + API NASA
4.Появляется лёгкий бэкенд-прокси (Cloudflare Workers / Vercel Functions)
5.Появляется поиск по городу + автодополнение

5.Используемые технологии

·  HTML5
·  CSS3 (flex, grid, clamp(), custom properties)
·  Vanilla JavaScript (ES2022+)
·  Шрифты: Google Fonts (Inter / Roboto / Manrope / ...) или локальные
·  Хостинг: Vercel / Netlify / Cloudflare Pages
·  Система контроля версий: Git + GitHub
·  Авто-деплой: push в main → мгновенный деплой
·  Редактор: VS Code
·  Тестирование: просто открываю в Chrome DevTools + на телефоне


6.Инструкция по запуску

Самый простой способ: кликнуть на эту надпись или в браузере (Google Chrome, Safari, MS Edge, Yandex) ввести адрес https://arbc.space
